  • What is XSS? - Cross-Site Scripting (XSS) is a type of vulnerability that allows attackers to inject malicious scripts into content from otherwise trusted websites. It exploits the trust a user has in a website.
  • Types of XSS:
    • Reflected: Script is reflected off a web server, such as in an error message or search result.
    • Stored: Script is permanently stored on the target server (e.g., in a database).
    • DOM-based: The vulnerability exists in client-side code rather than server-side.
  • Payload Basics: Learn how to use alert() for testing, and inject via tags like script, img, svg.
  • Contextual Execution: The execution of XSS depends on the context: HTML body, attribute, script, or URL â€” each reacts differently to payloads.
