üîç What is XSS?

Cross-Site Scripting (XSS) is a security vulnerability that allows attackers to inject malicious scripts into web pages viewed by users. These scripts can steal session cookies, log keystrokes, redirect users to malicious websites, and perform actions on behalf of the user. XSS attacks exploit the trust a user places in a website or the website places in the user‚Äôs input.

üß™ Types of XSS

  • Reflected XSS: Occurs when user-supplied data in HTTP requests (e.g., in a URL or form) is immediately returned by the server without proper validation or escaping. Commonly seen in search results or error messages.
  • Stored XSS: The malicious script is stored on the server (e.g., in a comment field or user profile) and is executed when users view the stored data.
  • DOM-Based XSS: A form of XSS where the vulnerability exists in client-side scripts. The server is not involved; the client-side JavaScript handles user input insecurely and updates the DOM directly.

üí° Payload Basics

Payloads are small code snippets injected into vulnerable points of a website to demonstrate or exploit XSS vulnerabilities. Common payloads include:

  • <script>alert('XSS')</script> ‚Äì classic script injection.
  • <img src=x onerror=alert('XSS')> ‚Äì uses image tag error handler to trigger JS.
  • <svg onload=alert('XSS')> ‚Äì another event-driven tag payload.
  • <script>alert('XSS')</script> ‚Äî The classic and simplest alert popup.
  • <img src=x onerror=alert('XSS')> ‚Äî Using image tags and error events to trigger JS.
  • <svg onload=alert('XSS')> ‚Äî SVG tags that run scripts when loaded.
  • <body onload=alert('XSS')> ‚Äî Body tag event triggers on page load.
  • <iframe src="javascript:alert('XSS')"></iframe> ‚Äî Iframe with JavaScript source (some browsers block).
  • <a href="javascript:alert('XSS')">Click Me</a> ‚Äî Using javascript: URI in links.
  • Event handlers like onmouseover, onclick, onfocus to trigger payloads in attributes.
  • Encoding payloads with HTML entities or URL encoding to bypass filters.
  • Using template literals and backticks in newer JavaScript environments.
  • Combining HTML injection with script injection to manipulate page content.
