  • What is XSS?
    Cross-Site Scripting (XSS) is a type of vulnerability that allows attackers to inject malicious scripts into content from otherwise trusted websites. It exploits the trust a user has in a website.
    XSS attacks can hijack sessions, deface websites, redirect users to malicious sites, or steal credentials.
  • Types of XSS:
    • Reflected: The script is reflected off a web server (e.g., in a URL parameter or search result). Usually not stored, but activated by a user clicking a malicious link.

      Lab 1: Reflected XSS in Search (Open Lab)
      Lab 2: Reflected XSS in Form Action (Open Lab)
      Writeup: These labs demonstrate how reflected inputs (query strings or form data) without sanitization can lead to script execution. Try payloads in search or form inputs.
    • Stored: The malicious script is permanently stored on the target server (e.g., in a comment or user profile) and runs whenever anyone views the page.

      Lab 3: Stored XSS via Comment Field (Open Lab)
      Lab 4: Stored XSS in Profile Bio (Open Lab)
      Writeup: These labs allow script injection in fields stored in a backend database. Reload pages to trigger stored scripts.
    • DOM-based: The vulnerability exists entirely on the client side and is triggered when JavaScript modifies the DOM based on untrusted user input.

      Lab 5: DOM XSS with location.hash (Open Lab)
      Lab 6: DOM XSS from innerHTML Injection (Open Lab)
      Writeup: These labs use JavaScript to parse or reflect URL hash values into the page DOM. Learn the difference between safe and unsafe DOM methods.
  • Payload Basics:
    Start with simple testing using alert() to see if a script is executed. Inject using tags like <script>, <img onerror>, and <svg onload>. These payloads help you understand execution contexts and browser behaviors.
  • Contextual Execution:
    Understand where the payload is being injected: in the body, attribute, URL, or inside a script block. Each context needs different escaping and payload strategy.

    Lab 7: HTML Attribute Context XSS (Open Lab)
    Lab 8: JavaScript Context Injection (Open Lab)
    Writeup: Learn how escaping affects execution depending on where the script appears in the HTML or JS structure.
